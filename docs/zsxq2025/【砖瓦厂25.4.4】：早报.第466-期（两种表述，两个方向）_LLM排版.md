
# 【砖瓦厂25.4.4】：早报.第466 期（两种表述，两个方向）

人的思维和认知的改变，是一切改变的基础。我持反对意见。

用MANUS 的方式，把这句话解构下，她表述的是：
△思维 or 认知→一切其它的△
△为变量。

令 f(x)为其它一切的状态
x 为思维或认知

那么每一个x, 有且只有一个唯一的f(x)与之对应。
这就是初中我们学的：函数的基本定义里的y=f(x)。

我们为了简化模型，把 f(x)定义为：薪资。当然，也可以定义为伴侣颜值、睡眠质量…
好了 薪资)是否会随着认知的变化，而必然变化呢?

且同一个x, 不能有>1个的f(x)出现
即同样的认知变化，不能有同样的薪资变化局面出现。

以及：薪资的变化，是严格要求建立在——认知的变化上。

所谓基础嘛——我们稍微思考下，可能会得出结论：薪资的变化，是多参数共同的函数，且认知不是其必要条件。

这些参数可能包括：运气、行业整体发展状况、认知、加班程度、年限 ……

比如在军队，薪资就是和年限强相关的
军官的薪资，是服役年限、职级、军衔、家属是否随军等多参数的函数。

我们令薪资为： y
那数学建模的情况一定是：
y=f(x1,x2,x3…)
x1 x2x3分别为服役年限、职级、军衔、家属是否随军等。

那为什么,很多人愿意这样讲：认知的改变，是一切改变的基础呢?
因为这样做：符合口语化下的简单表述习惯。
生活沟通时用，一点毛病没有。

但是，最好少用在偏严肃的场合：比如工作中。

举例子：十年前，你对着谷歌、百度的输入框说：我现在预算3000,想去西南旅行，三天路程，请帮我出一份攻略：这段表述中，用户输入的就是一个长文本的 x 。

大家要注意了
任何软件，都是一个封装了的大函数。
所谓各种模型，就是x 与 f(x)之间的映射各种关系。
各种网站模型就是： f1,f2……fn

十年前，我们会发现，当用户输入长文本的x 时
谷歌的、百度的模型，给出的 f1(x),f2(x)
大概率是乱七八糟的内容。
没准是让你莆田医院三日游。

而现在，函数进化到了fn
它可以是GPT, 可以是deepseek, 可以是MANUS 。
它们是可以处理长文本的x1………xn的。
并给出高质量的： f1(x1)……fn(x1)。

而这些函数里，嵌套了有：Transformer, 有注意力机制。
以及：非常强大的NLP (如切词，分词)技术。
大家知道的，谷歌早在二十年前，就开始了对这些技术的投入 。

下面是我想表达的关键部分了：
现代文明，似乎非常非常强调逻辑的完整性和通顺性
而人类，包括东方的古老文明，似乎更加偏袒：写意!
这两种思考、行为习惯，把人类族群，拉向了两种发展模式。

各有所长，各有所重。
前者的社会土壤，更加容易做出牛逼的企业。后者风格下，宜于粘合族群。

于是我们会发现，我们身边的牛人：
在做事上，逻辑性超强
做人上呢，又充满写意滋味的魅力

可反过来，就完全不同了。
若是一个人，在：
工作上，合作上，呈现出烂漫之态。
生活上，又一板一眼，以至于严苛。

你会发现，就有点上头了。
