
# 亚马逊逆向工作法：PR-FAQ式产品创新

## 新闻稿的内容结构

PR-FAQ 分成新闻稿和常见问题解答。新闻稿的篇幅通常不超过 1 页纸，常见问题解答不超过 5 页纸。新闻稿是负责给读者描绘精彩的产品体验；而常见问题负责解释产品体验背后的技术和资源细节，并评估公司将要为新产品面对多大的成本与挑战。

新闻稿虽然短小精悍，但通常要包含 6 个必要部分：

1. **标题**。用一句话，以目标客户容易理解的方式，点出产品的名字。
2. **副标题**。也是用一句话，描述产品以及它带给客户的好处。
3. **摘要**。这也是新闻稿的第一段，主要把产品发布的时间地点、发布者写清楚。稍微再把产品的差异功能描述一下。
4. **进入重点了：站在客户立场，描述新产品要解决的具体问题**。
5. **解决方案：告诉顾客，你的新产品是怎么解决前面这种棘手问题的**。
6. **引用及购买**。新闻稿的最后，应当先引用公司发言人的一句话，或者假想顾客的一句话，来描绘用上新产品之后的便捷场景。这其实是在引导潜在顾客，把自己的美好生活跟当前这个新产品给勾联起来。做完这个勾联之后，再给出购买链接或者沟通渠道，让动了心的顾客可以继续了解、直接下单。

## 常见问题的内容结构

常见问题部分的结构，不像新闻稿那么固定，但也有它的必要组成部分。

首先要分成两大类，一类问题更多是来自顾客、媒体等外部声音的，这叫“对外常见问题”；另一类更多是来自公司内部各个部门的，像财务、法律、物流、运营，这叫“对内常见问题”。

对外常见问题，常见的就是产品的 工作方式、定价、怎么买、售后等等。其实不需要单独列出来，把它们揉进新闻稿的正文里就可以了。因为新产品所处的市场不同、面对的用户不同、置身的时代也不同，用户的关切也会非常个性化，这就非常考验产品团队的市场洞察能力。

常见问题里，更值得认真对待的是对内常见问题。亚马逊认为有几类通用的标准化问题，是都需要考虑的。

1. **关于客户需求与潜在市场规模的论证**。
2. **经济效益与成本结构，比如产品的毛利、投入、定价逻辑**。
3. **如何让产业链上下游愿意接纳新产品**。
4. **技术、工程、财务、法律等各方面的可行性问题**。

在大原则上，外部常见问题追问的是客户会不会买、跟竞品有什么不同；内部常见问题追问的是技术可不可行、成本能不能承受、产业链会不会配合、账算不算得过。

## Kindle 的开发

这样一份 6 页纸的 PR-FAQ 写完之后，亚马逊的产品团队就要拿着它，去跟公司高层反复开会，收集新的问题，再去修改和迭代 PR-FAQ，来理顺开发思路，让想法真正变得靠谱。

20 年下来，亚马逊内部每年虽然都会诞生几百个 PR-FAQ，但多数都不能真正立项开发、进入市场。这其实不是浪费精力吗？亚马逊不这么想。在这家公司看来：

首先，大多数 PR-FAQ 没能落地，其实 是帮亚马逊降低了试错成本、把资源真正集中到那些有最大把握、最大影响力的产品上。其次，团队迭代 PR-FAQ 的过程就像修改论文，对新产品的审议者和开发者来说，都能更深切地理解市场、理解产业、理解公司，让他们的能力得到极好的提升。

亚马逊著名的电子阅读器 Kindle，就是从一大批 PR-FAQ 文档里杀出来的。

这个项目在 2005 年前后开始启动。最早做产品规划时，还是沿用 PPT 思维，也没有客户意识。当时，乔布斯约贝索斯和亚马逊团队进行过一次会面，告诉亚马逊高管们，苹果将要推动的数字音乐革命，将会把音乐 CD 挤出历史，亚马逊的 CD 销售业务将会极大萎缩。

这事把亚马逊给刺激到了。所以，研发团队规划数字媒体业务的时候，想过音乐硬件，也想过阅读硬件，但考虑的都是做一个怎样的产品，能更有利于公司，而不是客户。即便明确了要做电子阅读器，也更多是从技术挑战、商业约束、销售、财务前景、市场机会层面去论证这件事该做。

但当研发团队被逼着去给 Kindle 写新闻稿之后，他们不得不面对拷问，去关心如何实现客户真正想要的东西。比如，如何确保用户愿意为电子书付 费而不是盗版？怎么让阅读体验真正 超越纸质书？

在这些以终为始的问题倒逼下，研发 团队几经修改，终于把模糊的构想落实为可执行方案，确定了 Kindle 这个新产品的核心体验目标：让用户在 60 秒内购买并开始阅读一本书。

这句话看似简单的表述，后来成了 Kindle 产品的北极星目标。硬件团队 围绕它来确定无线方案，内容团队 围绕它来谈版权合作，出版部门 围绕它来同步上线电子书，供应链也围绕它 做调整。换句话说，这句话，成了亚马孙逆向定义 Kindle 这个产品的锚 点。

2007 年，Kindle 正式发布，确实实现了让用户一分钟内买好书开始阅读的 北极星目标，也成了亚马逊的长红产品。后来有人形容：Kindle 不是工程师设计出来的，而是“从未来顾客的心里被写出来的”。这句话，几乎精确概括了亚马逊逆向工作法的精神。

## 总结

这一讲，我们了解了亚马逊著名的 PR-FAQ 工作流程。你有没有觉得，它不只是一种产品立项流程，更是一种文化仪式。它让亚马逊在成长为百万员工规模的同时，依然保留了创业公司式的专注与敏捷。

当所有人都以未来顾客体验为起点倒 推资源，协作就不再靠命令，而是靠 目标。这也是为什么在亚马逊内部，写好一份 PR-FAQ 的人，更容易获得 跨团队的信任与话语权，因为那份文 档让所有人都看见了“我们要去的地方”。

支撑 PR-FAQ 的逆向工作法，其实也 是我们生活中常见的一种决策哲学： 先定义我们想成为怎样的人、过怎样 的生活，再反推今天应该有怎样的选择和行动。

可以说，它跟斯多葛哲学正好相反， 一种是系统设计型理性，一种是顺势 自洽型理性。前者让我们去创造世 界，后者让我们不被世界摧毁。今天 的你，更向往、更享受哪一种呢？
