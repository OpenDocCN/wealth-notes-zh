需求分析：给我的AI客服立下三条铁律
我的小店主营成人用品（男性延时、飞机杯），属于所谓的“黑五类”，客户问题非常集中，且平台监管严格。因此，我的AI客服绝不能信马由缰，必须满足以下几个苛刻的需求：
绝对服从：客户80%的问题都集中在使用方法与效果上。AI必须严格依据我提供的知识库作答，一个字都不能自由发挥，避免合规风险。
谨言慎行：产品特殊，存在大量违禁词和高危词。AI回复时必须能自动检测并规避这些词语，确保安全。
图文并茂：部分回复（如用法指导）需要以图片形式发送，脚本必须具备发送图片的能力。

开发哲学：先造“独轮车”，再改“方程式赛车”
这里想分享一个最重要的心得：千万不要上来就想做一个完美的软件！AI目前还无法一步到位理解你的终极复杂需求。正确的做法是，先让AI帮你造一个能跑的“独轮车”，再慢慢给它加轮子、装发动机，最终变成你想要的汽车。在原有版本上不断优化，远比推倒重来要高效得多。
我的版本迭代之路：
V1.0 (独轮车版): 接收消息 -> 调用知识库 -> 回复消息 (能跑就行)
V2.0 (自行车版): 接收消息 -> 判断售前/售后 -> 售前查库回复 / 售后飞书提醒 (增加了分支判断)

最终版 (汽车版) 核心逻辑：
连接浏览器：自动接管一个已登录抖店后台的浏览器。
监控新消息：像雷达一样，循环扫描左侧列表，寻找“<30秒”的新消息提示。
智能处理文本：
规则优先：先用关键词匹配简单的固定回复（如发图、打招呼）。
AI兜底：规则匹配不上，则抓取上下文，调用Coze API进行智能回复。
内容审查：AI回复后，再进行一轮关键词过滤和替换，确保安全。
处理非文本消息：如果用户发的是图片、表情，脚本会判断为“无法处理”，并立即发送一个HTTP请求，通过外部服务提醒我人工介入。
防重机制：处理过的消息会打上“已读”标记，绝不重复回复，避免骚扰客户。

开发之前要考虑的问题：
在敲下第一行代码（或者说，写下第一个Prompt）之前，先别急。花点时间思考清楚以下几个问题，会让你的开发过程事半功倍。更妙的是，这些问题你完全可以丢给AI，让它帮你一起规划：
技术路径：我们要用什么工具来实现？（例如：Python + Playwright库，还是其他方案？各有什么优缺点？）
运行环境：脚本将来在哪里跑？（是在你自己的Windows电脑上，还是需要一台云服务器？）
工作流程：把整个任务拆解成一步步的动作。（例如：1. 打开网页 -> 2. 输入账号密码 -> 3. 等待新消息... 越详细越好）

你可以像这样问AI，让它帮你做“军师”：
我没有编程基础和经验的小白，现在我想开发一个脚本或者软件，实现（自动回复抖店客户的消息），我设想的思路是：首先获取到客户的消息，然后使用api提交给AI大模型，获取到AI大模型的回复以后，再发送给客户，我该怎么去实现，先帮我设计一下，不需要给出具体的代码

import re
async def main(args):
# 1. 获取完整的输入文本
content = args.params.get('input', '')
# 2. 初始化默认值
product_name = "" # 初始化为空字符串，以便直接拼接
customer_question = "未识别到问题"
# 3. 提取商品信息
product_pattern = r'(?:最近订单|咨询商品): (.*)'
product_match = re.search(product_pattern, content)
if product_match:
    full_product_info = product_match.group(1).strip()
    product_name = full_product_info.split('(')[0].strip()
# 4. 提取客户最新问题
question_pattern = r'---客户最新问题---\s*(.*)'
question_match = re.search(question_pattern, content, re.DOTALL)
if question_match:
    customer_question = question_match.group(1).strip()

# 5. --- 修改点在这里 ---
# 直接将商品名和问题拼接成一个句子
final_sentence = f"{product_name}{customer_question}"
# 6. 返回拼接后的最终结果
# 您需要在节点的【输出】部分，添加一个名为 'final_sentence' 的参数
return {'final_sentence': final_sentence}

1. 遇到不会的问题，表达歉意并告知客户可以查看商品详情页。
处理特殊情况
1. 若客户声称使用产品后有副作用或不良反应，坚持产品经过检测，无不良成分，不会造成负面影响。

- 严禁承认产品标题、图片等存在误导或虚假宣传。

脚本开发的详细步骤（多图预警，此处包含详细的与AI对话的截图，并附解释）
最初我的脚本是使用的油猴实现的，后面油猴存在着局限性，才转了Python，这是当初的提问，我是从获取页面元素（注：获取元素，个人理解就是让脚本去获取网页上的内容，网页就像一个酒店，内容存在不同的房间，我们获取网页元素，就可以让脚本直接去对应的房间获取到对应的内容，比如我们需要的客户消息、发送按钮等等）存在全部由AI指导完成，我只是负责一步一步跟着去操作，然后将结果发给AI

在开发过程中，我是将每一步拆解开，然后最终再组合，这是考虑到AI目前的记忆力和专注力都有限，每次对话如果是让他从一大堆代码中修改一小部分，偶尔会出错，而且整个过程回顾时太复杂，比如我们要做人形机器人，需要她走路，那么我们平常是怎么走路的？一般都是先迈右脚，再迈左脚，以此交替，迈右脚又可以拆解成先抬脚，向前平移，再落脚（当然机器人肯定没这么简单，重点是拆解的思路）

即使遇到问题了也是很正常的，把问题告诉AI就可以
所有问题一股脑的告诉AI就可以，此处必须表扬Gemini，还会用大白话讲解一些术语，属于是边干边学了（虽然也记不住）

测试的思路是很重要的，我们完成了一个功能，就要去测试，而不是急于开发下一个功能，比如这里，AI告诉我脚本的功能已经可以了，直接为我提供了一个成品脚本，但是我们并不确定是否真的可行，因此为了避免无用功，先测试一下是否可以工作

有BUG就告诉AI去解决，不要灰心
当我们需要加入新功能的时候，直接告诉AI我们的需求，然后按照他说的去做就可以了，继续重复上述步骤

勤测试，勤测试，勤测试，这是很关键的一步，可以避免你不知道问题出在哪里
在开发过程中，我们要不断去提问题和挑错，虽然我们不懂编程，但是我们都是各个软件的用户，对于软件挑毛病是很在行的
