
# 从零到一的首单复盘：一个普通程序员的105天淘金之路

## 写在前面

这是我作为 Web 出海新人，从零开始做海外产品 3 个月的复盘笔记。从一个只懂安全底层开发的程序员，到打通需求-产品-流量的完整闭环并获得首单，这 105 天的经历让我完成了从技术视角到创业视角的彻底转变。整个过程充满了运气成分，但更多是方法论积累和认知的迭代。希望这篇复盘能为同样在出海路上摸索的朋友提供一些参考。

## 新手村：构建全局认知

## 初始认知的局限

一开始，我对海外产品的认知极其有限，仅停留在"找词→上站→SEO"的简单理解。花了整整一个月时间在家自学，看航海资料、记笔记、补技术栈，因为要学的东西实在太多。整个出海闭环其实就三个核心环节：需求、产品、流量。

## 需求挖掘：从厚到薄的方法论

需求层面的所有方法都来源于航海手册和星球精华帖——看榜单、用词根找词、站找词、词找站等。关键在于不断实践每条方法论，持续复盘，提升需求洞察的认知和体感。

这个过程需要先把书读厚，再把书读薄：读厚：建立全面认知，掌握十几种方法论；读薄：通过实践找出最重要的切入点，形成路径依赖。否则航海手册里的方法太多，会在实践中搞不清重点，把时间浪费在不重要的地方。

在积累找词词根、熟悉各种方法论的同时，还需要掌握工具使用：AITDK、SimilarWeb、SEMrush 等。

## 产品开发：速通式技术补强

因为我的技术栈偏底层（安全方向，Windows 客户端开发），前端后端数据库基本都得重新学。我采用了速通方法论：

- 快速预览：每个新技术栈先花 10 分钟看 B 站速通视频，了解大概
- 整体刷新：前端所有技术栈（TailwindCSS、React、Supabase 等）两小时刷完
- 实战补课：遇到具体问题时再回来补知识点

这种方法能最大程度节省学习成本。在 Vibe Coding 过程中，很多语法细节根本用不到，没必要全部按部就班学一遍。

## 通过这种方式，可以在最短时间内补齐初级前端项目所需知识，在出海前期完全够用。因为前期遇到的项目大多数都是 mvp 开发，套用模板，配齐登陆支付，接 api，所有各个功能相互独立，互不干扰，对于新手来说，掌握基本的编程知识完全够用。
