
# Claude Skills 从0到1实战教程，一个案例带你入门（小白友好）

AI 无法直接“看”视频，这让它错过了 B 站等平台里大量硬核、优质的信息。这篇文章解决的就是这个问题。作者将一步步带你构建一个 AI 技能（Claude Skill），让它能自动抓取视频字幕，分析内容，并在 10 分钟内生成一份结构化的选购指南或研究报告。文章记录了完整的 Skills 制作过程：从分析需求、让 Claude Code 写脚本、解决 Cookie 问题、优化字幕格式、到最终封装成可复用的技能包。读者将看到如何将一个复杂的人类任务，拆解并翻译成 AI 可执行的工作流，并最终固化成一个可随时复用的“技能包”。

## 人是怎么买东西的

在动手之前，咱们先想清楚一个问题：如果是人来干这件事，流程是什么？这很重要，因为让 AI 做事，本质上就是把人的工作流程翻译成 AI 能理解的步骤。人买东西的流程大概是这样：

1. 打开 B 站，登录账号。
2. 搜索「加湿器测评」，看看哪些视频播放量高、标题看起来靠谱、UP 主是不是专业做测评的。
3. 点开几个视频，完整看完，边看边记笔记：这款容量多大、那款噪音怎么样、哪个品牌容易坏……
4. 综合自己的需求（比如房间大小、预算、有没有特殊功能需求）和视频里学到的信息，筛选出几个备选项。
5. 打开某宝、某东去比价，看看哪家店在做活动，最后下单。

整个流程，核心就是：搜索 → 看视频 → 学习信息 → 做决策。那让 AI 来做，流程也应该是类似的。但有个关键问题：AI 不能直接看视频。虽然目前的 Gemini 确实有看视频的能力，但效果一般，而且一个视频占用 token 太多了，也无法做到一次性“看完”好几个视频。AI 能理解的是文字，你给它一个视频文件，它打不开，就算有些 AI 说能“看视频”，实际上也是先把视频转成文字描述，再理解内容。

## AI 要“看”视频，得先把视频的内容转成文字，也就是字幕

所以，AI 要“看”视频，得先把视频的内容转成文字，也就是字幕。这就是第一步要解决的问题：怎么让 AI 获取到 B 站视频的字幕？

## 写个脚本比打开浏览器更优雅

你可能会想：那就让 AI 操控浏览器，打开 B 站，搜索视频，然后用第三方插件获取字幕不就行了？这个思路没错，技术上确实可行，但有几个问题：

- 信息太多了：打开浏览器访问 B 站，页面上的东西太杂了，广告、推荐视频、评论、标题、侧边栏……这些东西对人来说可以自动忽略，但对 AI 来说都是“信息”。
- 需要占用我的电脑：让 AI 操控浏览器，意味着它要在我的电脑上打开一个浏览器窗口，这个过程我没法在后台运行，得一直等着它操作完。而且，这个浏览器窗口是我的电脑在跑，AI 本身没有自己的浏览器环境。

## 所以，更好的办法是：直接写个脚本去获取字幕

B 站的视频字幕，其实是存在 B 站服务器上的一个文件。我们可以写一个简单 Python 脚本，直接调用 B 站的接口，把字幕文件下载下来。这样做的好处是：

- 干净，脚本只获取字幕，不会加载页面上其他的 内容。
- 快，不需要等浏览器加载、渲染页面。
- 静默，在后台运行，不占用我的屏幕。

## Claude Code 写字幕脚本

如果你还不了解 Claude Code，我简单介绍一下：它不是一个只能聊天的 AI，而是一个能直接干活的 AI Agent，运行在你的电脑终端里。你可以理解成一个“会写代码、会自己运行代码、会浏览网页、会读写本地文件”的 AI 助手。它最大的不同在于，你给它代码，它不只是给你看，而是会自己去执行和调试。它可以：

- 自己写代码，写完了自己运行，看看结果对不对
- 访问你的本地文件，读取文件、修改文件、创建新文件
- 上网搜索信息，遇到不懂的技术问题自己去找
- 不断调试，代码跑不通就自己改，改完再测，直到成功

## 这就很适合我现在的需求：我不想自己去研究 B 站 API 怎么用，我只想告诉 AI

“我要获取 B 站视频字幕，要装什么脚本、什么插件你自己找”，然后让它自己搞定。所以我打开 Claude Code，直接给它提了个需求：

我需要一个东西能搜到 B 站的视频，获取 B 站视频网址，然后如果这个视频有字幕的话，自动帮我提取。自己想办法解决，给你了互联网搜索权限、MCP 权限了，把代码什么的都放到当前目录，然后去测试。

Claude Code 开始工作了。它先上网搜了一下“B 站 API 获取字幕”，找到了一些技术文档和别人写的代码示例。然后它开始写代码，创建了一个 Python 文件，写了请求 B 站接口的逻辑，写了解析字幕数据的函数。

## 写完之后，它自己运行了一下，测试能不能用

第一次测试失败了。代码跑起来了，但没成功。Claude Code 告诉我：获取不到字幕，可能是这个视频没有字幕。但我知道这个视频肯定有字幕，我自己在浏览器打开这个视频，字幕按钮是亮的，点开就能看到字幕。所以我回复它：“这个视频绝对有字幕，你再看看是不是哪里的问题”。

## Claude Code 又开始捣鼓

它重新检查了一遍代码，又去搜了一下 B 站 API 的文档，然后发现了问题：要获取 B 站视频的字幕，必须提供 Cookie。

## Cookie 是什么

如果你不懂编程，可能不太清楚 Cookie 是什么，我简单解释一下。Cookie 是网站用来“记住你”的一串字符。你登录 B 站后，浏览器会保存一个 Cookie。下次你再打开 B 站，浏览器会自动把这个 Cookie 发给 B 站的服务器。B 站服务器一看：“哦，这个 Cookie 我认识，这是用户 xxx，已经登录过了。”然后就不会让你重新登录。Cookie 就像一张“通行证”。有了它，网站就知道你是谁、你有没有登录、你有什么权限。那为什么获取字幕需要 Cookie 呢？因为 B 站有反爬虫机制，如果一个请求没有 Cookie，B 站的服务器会认为：“这不是正常用户在看视频，这是个爬虫在抓数据。”然后就拒绝返回字幕。但如果请求里带上了 Cookie，B 站就会认为：“这是一个已登录的用户在看视频。”然后就正常返回字幕。所以，我得给 Claude Code 提供一个 Cookie。

## 怎么获取 Cookie

获取 Cookie 很简单：

1. 打开浏览器，登录 B 站，我用的 是自己的 B 站小号（不用大号是怕被封号，虽然这个风险很小）。
2. 按 F12 键，打开浏览器的“开发者工具”。
3. 点击“Application”标签（应用），然后点击“Cookies”，找到 bilibili 的页面，把这里的“name”和“value”的值复制粘贴下来。

## 你不用管到底哪一段 Cookie 才是有用的，不需要你背后的原理

直接把所有都 input 给 Claude，它会自己判断哪一段是有用的，Claude Code 把 Cookie 加到了代码里，再次运行。这次成功了。终端输出了字幕内容，我看了一眼，确实是视频里的字幕，没问题。到这里，第一步就完成了：我们有了一个能获取 B 站视频字幕的脚本。

## 优化字幕格式

脚本能用了，但还有个问题。我让 Claude Code 打开获取到的字幕文件看了一眼，发现是 SRT 格式，长这样：

```
1
00:00:01,000 --> 00:00:03,500
大家好，今天测评加湿器
2
00:00:03,500 --> 00:00:06,000
我们准备了五款不同价位的产品
3
00:00:06,000 --> 00:00:09,000
首先看第一款，某品牌的入门级产品
```

每一段字幕前面都有序号，还有时间戳（00:00:01,000 这种）。这些信息对人看视频有用，你知道这句话出现在视频的第几秒，可以快速跳转，但对 AI 来说，这些就是噪音。AI 要理解的是字幕的内容，序号和时间戳不仅没用，还会占用 token。

这就是上下文工程的一个关键原则：确保给 AI 的输入信息密度是高的。去掉无效信息，有两个好处：

- 节省成本。每个 token 都要花钱，无效信息越多，成本越高。
- 提高质量。AI 的上下文窗口是有限的，塞了太多无效信息，真正重要的内容就被稀释了。

所以我让 Claude Code 修改脚本，把 SRT 格式转成纯文本的 Markdown 格式，删掉时间戳和序号，只保留字幕内容：

```
我需要你把字幕转成 Markdown 格式，修改时间戳格式、删掉字幕序号，只保留文字内容，这样更适合 AI 阅读。
```

Claude Code 很快改好了，现在获取到的字幕是这样的：

```
00:01 大家好，今天测评加湿器
00:03 我们准备了五款不同价位的产品
00:06 首先看第一款，某品牌的入门级产品
```

干净、清爽、信息密度高。

## 如果你对上下文工程不太了解，想知道为什么要这样优化输入内容，可以看我之前写的文章

现在脚本能用了，字幕格式也优化了，接下来要验证一个核心问题：AI 读完这些字幕后，真的能给出靠谱的选购建议吗？这个问题很关键，如果 AI 读完字幕还是一头雾水，或者给的建议不靠谱，那整套流程就没意义了。所以我在当前对话里直接测试。我让 Claude Code 用刚才写的脚本，获取几个加湿器测评视频的字幕，然后我跟它说：

```
基于这些字幕内容，给我一份选购建议。
```

## Claude Code 开始工作，它读完字幕后，输出了一份报告，内容大概是

```
根据房间面积，XX 平米以下选 XX 升，XX 平米以上选 XX 升
某几个品牌的噪音表现好，适合放卧室
有些品牌的水箱设计不合理，容易漏水
XX 品牌和 XX 品牌价格相近，但 XX 品牌功能更多
```

我看完之后，发现这个建议是靠谱的。

## AI 确实能从字幕里提取出关键信息：哪些功能重要、哪些品牌性价比高、要避开哪些坑，然后给出结构化的建议

而且我发现一个有意思的点：我没有写复杂的提示词，AI 就能理解任务。我只是说“基于这些字幕给我选购建议”，AI 就知道要总结优缺点、对比不同产品、考虑我的需求。可能是因为“看测评然后给选购建议”这个任务本身很符合人类的直觉，AI 也能自然地理解。

到这一步，其实在这个对话窗口里，我已经能让 AI 帮我完成选购决策了。但问题是：下次我想买别的东西，还得重复这一遍流程。我得重新解释任务、重新让 AI 调用脚本、重新交代要求，就像每次都要培训一个新实习生，很累。这就是 Claude Skills 要解决的问题。

## Skills 就是把流程固化下来

Claude Skills 是什么？简单说，就是提前准备好的“技能包”。一个 Skills 里可以包含：

- 提示词：告诉 AI 怎么干活
- 参考文档：比如操作规范、格式要求
- 代码脚本：让 AI 直接调用，不用每次重新写

你把这些东西打包成一个文件夹，给到 AI ，以后每次对话，AI 都能自动调用这个“技能”，不用你重复解释。AI 会自己判断什么时候该用哪个 Skills。

你不用输入什么指令，只要说“帮我看看加湿器的测评”，AI 看到关键词，自己就知道调用“B 站测评分析”这个 Skills。
