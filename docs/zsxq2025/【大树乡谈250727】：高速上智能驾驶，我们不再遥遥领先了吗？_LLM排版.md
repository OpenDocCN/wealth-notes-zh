重点说下，为什么小镇说这次测试方案不严谨，甚至可以说，猫腻很多。比如视觉识别这块，可以看到测试用的假人竟然是白色的，那么为什么用白色而不是黑色呢?原因很简单，白色在夜间，是最容易被纯视觉方案识别出来的，如果把测试假人换成黑色、紫色，跟夜间背景色几乎一致，甚至把测试安排在雨夜或者人为制造迷雾，纯视觉方案就会严重暴露无法克服的技术短板。

特斯拉在美国就有很多事故案例，已经充分证明了纯视觉方案无法避免的识别问题，尤其难以识别颜色相同、相近的背景和前景，导致事故发生。可以观察，纯视觉方案的车企，在进行夜间假人测试的时候，使用的假人颜色是什么。

其实测试跟问卷调研类似。比如美国去年大选，不同机构的民调数据相差甚大，这并不是造假，而是通过问题设计和采访对象选择，就能够实现。智驾测试也一样，测试方有无数种办法，可以让测试显得似乎很公平，实际拉偏架到了极致。尤其这次测试，刻意不去测纯视觉方案弱项的地方，几乎所有的测试场景设置，都是确保纯视觉方案行得通，纯视觉方案肯定不行的，就不去设置，在这种情况下，如果纯视觉方案还不行，那才叫有鬼。而这种设置，相关企业也没法去起诉，因此不予置评反而是理性选择。总不能下场说方案设置有问题吧。

合理的测试，应该更加全面，要针对不同的技术方案全面测试。比如至少要增加，适合多雷达模式但相对于纯视觉有优势的场景，比如深夜、雨夜、大雾、纯黑色假人等等。

小镇还是之前的观点。目前特斯拉在智驾上大概还能保持一线水平，但纯视觉方案的技术短板决定了，特斯拉如果不赶紧进行改革，开始探索安装激光雷达、相控阵雷达、米波雷达等多种雷达的复合感知，未来在智驾领域，特斯拉将无可避免的跌入二线。

这不是危言耸听。

智驾好不好，基础是感知数据的准确性。纯视觉方案，获得的数据无可避免的粗糙，要想克服、提高安全性，就必须加大车载算力，就算把算力加到极致，也不可能完全解决感知数据粗糙的问题。更何况，哪怕不考虑成本问题，车载算力也是有上限的。

而加装了激光雷达等多种雷达的复合模式，目前的难点是在不同数据之间如何整合决策的问题，但这完全可以克服。多种感知可以最大限度确保感知数据的准确性，这就意味着，对车载算力的需求下降。

打个比方。

纯视觉方案需要10的算力，复合感知只需要1的算力。

那么即使不考虑安全性这个至关重要的问题，仅仅从经济性角度考虑。

纯视觉方案和复合感知方案，在经济上，就变成了算力芯片成本和雷达成本之间的比较。

算力芯片数量的叠加是有限度的，而提高单块芯片的性能，必然意味着成本激增；与之相对，激光雷达等制造成本可以更大限度下降，这几年已经证明了，随着中国产业链的持续迭代，未来激光雷达的成本必然远低于堆叠算力芯片。

仅仅从经济性上考虑，作为极限压榨成本代表的特斯拉，也应该选择安装激光雷达，从而降低车载算力芯片的成本。当然，目前特斯拉的难点在于，被限制充分利用中国产业链，而美国本土激光雷达等价格高得惊人，这才无奈的选择纯视觉。
