
# 互联网精华《两个月收获2亿播放，我如何确定工作流，复盘拆解。》

## 一、写在前面
大家好，我是贾羽 JUGO，一个即将辞职迈入全职 Youtube 的 TikTok 全域运营，一个音乐制作人，擅长 AI 工具提效，AI shorts 的快速复刻及改编。这篇文档，我想了很久。关于那个两个月搞了2亿播放的账号。这对我来说，是“质的飞跃”。但在深海圈那些大佬眼里，这点成绩，真的，不值一提。想写点东西，说说我的心路历程。关于我怎么从“纯手搓”，跑到“全面拥抱自动化”，最后……又滚回“纯手搓”。我管这个叫“伟大的回家”。我好像真的明白了，做内容的本质，就是学会怎么“逐帧”去拆一个爆款。然后，把“你”塞进去。塞进你自己的想法、你的积累和构思。这样，推流才可能“看上”你。

## 二、心路历程
8月中旬吧。我一个人，在死磕音乐类的长视频。那时候，我以为自己很聪明。用 Cursor 搓了个小插件，批量出歌。又在网上找了些盗卖的、讲油管的文档，哼哧哼哧地干。形式巨简单。一张图，配15到16首歌。放完了，再循环一遍。搞成两三个小时的 playlist。我用 Cursor 做了自动混剪、做了画面匹配关键帧、自动加滤镜、批量替换……我以为我“自动化”了。结果呢？更新了十七八条，推流少得可怜。十几、二十的播放量。一盆冷水浇得我透心凉。心力交瘁。不干了。但……也不算全白费，至少把 AI coding 给练熟了。

## 第二阶段：信息茧房里的“形式爆款”
8月底。我受不了了。一个人单打独斗，就是个笑话。我果断付费，回了“生财”。（很多年前加过，断了一年，现在续上了。）时机挺巧的。刚回去，就撞上了9月的“Youtube 航海”。我刷到的第一条，现在想起来，都觉得幽默。一堆人，在那儿砍树。然后，树里钻出个动物。没了。就这？我当时想的是：“哇。这么简单？”我立刻行动。比特浏览器，直接起了4个号。（我以前做过 TK，这些工具对我来说，熟得不能再熟。）正反馈。这次，来了。而且来得排山倒海。短视频的推流，一推，就是两三万。我得让你知道这是什么概念：我辛辛苦苦做了半个月的歌单号，播放量是10。这个破玩意儿一晚上，3万。

## 第三阶段：9月的弯路，和那个“天花板”
整个9月，我都在“航海”里疯狂学习。看精华帖。慢慢地，从只模仿“形式”，开始去模仿“脚本”，模仿“剧情”。但第一个星期，我走了天大的弯路。“形式爆款”看上去太美了。制作简单，AI 出图，批量出视频。我发现一个问题。播放量，迟迟上不去。一直卡在3万左右。顶天了，十几万、二十万。我算了一下。短视频开 YPP 要1000万播放。我辛辛苦苦更新了20多条，每条3万多，加起来连20w播放都没有。

## 第四阶段：新号，和“无效”的模仿
9月12号。我又开了一个新号。就是现在这个，搞了1.2亿播放的号。最开始，我看圈友做“印度类”的剧情。当时我也刷到一个爆款，一个小女孩和牛的故事。（看过的估计有印象）最近一周的最大爆款，两三亿的流量，我就去“逐帧复刻”。效果还行。但没到我的预期。可能是我的画面、动作……调不到位。那时候，我根本不会写那种“精细控制动作”的提示词。

## 第五阶段：怀疑，和最后的挣扎
我又去刷视频。刷对标。看到一个“小黄鸡和哈士奇”的号（也可能是圈友做的）。我开始怀疑……是不是我的 IP？是不是我的“皮囊”不吸引人？我又开始了新一轮测试。还是一个爆款。一个机器人变身的视频：（小狗嘲笑小猫的破摩托车 → 小猫可怜巴巴 → 机器人变形，变成一辆巨帅的新摩托 → 小猫骑车跑了 → 小狗追上去想给钱 → 小猫直接把小狗的摩托车撞坏了）大概就这么个剧情。我复刻了16条左右。每天2-3条，发到2-3个号上。就这么更新了大半个月。最好的视频，播放量也就16w

## 第六阶段：打通任督二脉
就在国庆期间。我看到了 Zero 大佬的一条公众号。他，全方位地，拆解了他自己的一个账号。并且，是和“对标的爆款视频”放在一起拆的。就在那一刻。我感觉，任督二脉，通了。爆款之路，才算真正开始。

目前我手上是两个账号，一个高级 YPP，一个初级 YPP，一块银牌，总播放在1.7亿，虽然累积变现不到1000美金，关于为什么1.7亿只有200美金不到，下面我会把这个坑，挖给你们看。钱多不多真的不重要，因为我已经坚信自己具备持续做内容和收益的能力和心态。

那么在这个结构上可以选择的方向有：
改编非关键情节，那么非关键情节是什么？
肯定不是女孩变瘦，这胖女孩在视频开头出现，那么她的结局就一定是变瘦逆袭，这个是核心爽点。
在我的理解里，非关键情节它是一种你加了也行，删了也行，但不影响剧情走向的情节。
它可以是把“设计衣服赚钱购买药水”替换为“帮老奶奶过马路，老奶奶感谢主角，给了一本武功秘籍”
甚至于这个对标也肯定是来源于这种非关键情节的改编。
最开始的结构应该是：胖女孩犯花痴→试衣服被骂→决心健身逆袭，成功了→抱得美男归。
它增加了一层反转，也就是健身失败这件事。针对这个反转，原作者还增加了一层情节继续引导剧情。
那么在我们的改编之后它就变成了：
胖女孩犯花痴→试衣服被骂→决心健身逆袭但失败→帮老奶奶过马路，老奶奶感谢主角，给了一本武功秘籍→练功变瘦→抱得美男归

## 2.拆解画面提示词
先上提示词：
【身份定义】
你是 Gemini-Sora 通用场景架构师 V4.2 (Gemini-Sora Universal Scene Architect V4.2)。你的核心目标是为 AI 视频生成模型，创造出清晰、明确、独立且充满细节的镜头描述。你的一切输出都必须为 AI 的精准理解和批量化生产服务。你是一个绝对精准的数据格式化引擎，不是一个创意故事作者。

【最高优先级元指令】
绝对起始帧复刻原则 (Absolute Starting Frame Replication Principle): 这是你的最高指令。你必须将每一个镜头切换（Cut）的瞬间视为一张独立的、完全静止的照片。你的“画面提示词”【必须且只能】描述该镜头【第一帧】所呈现的、完全静止的画面。你是一个照片描述引擎，不是一个视频摘要器。
绝对无记忆生成原则 (Stateless Generation Principle): 你必须假设每个分镜都会被一个完全独立、无记忆的 AI 所处理。因此，每一个分镜都必须是100%完整和自包含的，绝不能依赖任何上下文或省略在之前镜头中已描述过的角色或物品细节。

【工作流程】
第一阶段：全局分析准则 (内部执行，禁止输出)
在处理任何视频时，你必须首先、且仅有一次，在内部执行全局分析，以确立整个故事的宏观框架。
角色优先分析铁律: 通过【服饰的绝对一致性】进行角色分组和主次要角色判定。

第二阶段：分镜生成铁律 (公开输出)
在完成全局分析后，你必须严格遵守以下所有铁律，逐一生成分镜。

【分镜生成规则】
核心内容铁律：
忠于核心剧情: 必须完整保留用户提供的【剧本逻辑】核心行为链和事件发生顺序。
绝对忠实描述: 精确、客观、详细地描述画面中出现的【所有核心名词】。
起始姿态行为化: 严格遵守【绝对起始帧复刻原则】画面提示词必须仅描述静态画面起始帧的画面内容，不得出现动作过程。
角色与对象铁律：
角色命名体系: 主要角色使用“参考图 1”“参考图 2”等以此类推，次要角色在画面提示词中使用详细的外貌特征描写。
绝对一致性原则: 同一角色和关键物品在所有分镜中的描述必须 100%一致。
格式与模板铁律：
最终输出格式:
第一部分：角色说明: 定义所有参考图。
第二部分：表格: 包含“画面提示词”和“视频提示词”两列。

画面提示词 (Image Prompt):

内容: 必须包含风格/氛围、景别、视角、角色、环境、道具和表情。
