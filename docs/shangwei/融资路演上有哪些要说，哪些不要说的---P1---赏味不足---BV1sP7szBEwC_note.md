# 融资路演上有哪些要说，哪些不要说的---P1---赏味不足---BV1sP7szBEwC

在本节课中，我们将要学习融资路演中应该表达的核心内容以及必须避免的常见误区。我们将通过分析典型错误，帮助你理解如何准备一场专业、可信的路演。

## 概述

融资路演是向投资者展示项目价值、争取资金支持的关键环节。然而，许多创业者在路演中容易陷入用力过猛的误区，导致陈述内容与融资需求严重脱节，反而降低了可信度。本节课程将拆解这些误区，并提供清晰的改进思路。

## 路演内容的常见误区

以下是创业者在融资路演PPT和陈述中经常犯的几个错误，这些错误会严重削弱项目的可信度。

### 1. 夸大其词，脱离实际

许多路演者喜欢在PPT中极力吹嘘项目的技术、团队或专利有多么“牛逼”，声称项目能“上天入地”。但一个核心矛盾随之出现：如果项目真的拥有如此高的技术壁垒和稀缺性，为何还需要融资？在投资者看来，这就像一张**SSR（特级稀有卡）** 的卡面，却标着**Normal Card（普通卡）** 的价格，逻辑上无法自洽。

**核心矛盾公式：**
`宣称的极高技术壁垒 + 宣称的庞大市场前景 ≠ 合理的早期小额融资需求`

上一节我们指出了夸大宣传的问题，本节中我们来看看与之紧密相关的第二个误区。

### 2. 融资额度与项目前景不匹配

路演者在前半部分将项目前景描述得无比宏大，仿佛需要千万级资金才能启动，但最后提出的融资需求却只有50万或100万。这种巨大的落差会让投资者产生疑问：你究竟是真的想做成这件事，还是仅仅为了骗取一笔小额资金？合理的融资额度应基于详实的业务规划和财务预测。

**错误示例：**
> 项目描述：“我们的AI机器人将颠覆万亿级市场，拥有独家专利。”
> 融资需求：“现寻求天使轮融资100万元人民币，出让10%股份。”

这种不匹配直接暴露了规划的不严谨。

### 3. 过度依赖“专家站台”

为了增加可信度，许多PPT喜欢罗列一堆专家、教授、主任医师甚至老外的头像和名字，仿佛在展示“家族图谱”。但在当前的投资环境下，这一招已经过时。资本方对此的看法很现实：这些“站台”的人，要么能**带资入场**，要么能带来实际的**客户资源（甲方爸爸）**。如果两者都不能，仅仅是一个名字，那么在投资者眼中，这无异于挂了个“此地钱多、人傻、速来”的牌子。

**资本逻辑：**
`专家价值 = 带资能力 或 资源变现能力`
`若上述值为零，则专家挂名 = 无效装饰 = 减分项`

### 4. 混淆“产品演示”与“可商用产品”

尤其是在AI、机器人、低空经济等软硬件结合的领域，做出一个**Demo（演示版）** 和做出一个稳定、可商用的产品，有着天壤之别。Demo可以很快做出来用于展示，但真正投入市场需要持续的开发、维护、迭代和庞大的团队支持，成本急剧上升。

**成本真相：**
`开发一个炫酷的Demo成本：X`
`开发并维护一个可商用、稳定迭代的产品成本：X * 10 (或更多)`

如果路演者声称用一个Demo就解决了行业痛点，却只申请一两百万的资金去完成后续所有工作，这听起来就像“过家家”，极不专业。

## 核心原则总结

本节课中我们一起学习了融资路演中四个“不要说”的核心误区：
1.  **不要**脱离实际地夸大技术与前景。
2.  **不要**让融资额度与描绘的宏伟蓝图严重脱节。
3.  **不要**堆砌无法带来实际资金或资源的“专家”名单。
4.  **不要**混淆产品演示版与成熟可商用产品之间的巨大成本鸿沟。

一场成功的路演，其核心在于**逻辑自洽**与**真诚可信**。你需要用扎实的计划、清晰的数据和合理的诉求，向投资者证明你不仅有一个好想法，更有能力将其实现。下一节，我们将探讨路演中“应该说什么”，即如何构建一个有力、可信的项目陈述。
