## 一、什么是社群转化模型

简单版本：将意向用户导到一个社群里面，运营一段时间，批量做转化产生营收的模式。

细致版本：狭义的理解为社群低转高的训练营转化模式，这里还是以我熟悉的知识付费产品举例，将购买了低价课产品的用户，拉到一个社群里面，进行3~14天（也有1~30天）的集体学习，具体学习形式：

- 输入：主要是看课程，分享形式可以是图文，可以是录播，可以是直播，可以是其中三者的混合。
- 输出：学员完成的事情，可以是写感悟，做作业，练习打卡，做测试。
- 氛围：社群要营造学习的氛围，一部分是带班老师（销售）要完成的事情，围绕学员输出过程和结果进行服务，例如学员做作业不会的问题需要答疑，学员做完作业之后老师进行点评；另一部分是同学们（水军，俗称托）完成的事情，交作业，提问题，甚至学员组队pk，目的就是塑造良好的学习氛围。

通过以上一系列的动作，提升用户的学习体验及意向度，塑造学员对品牌、课程、老师的信任度，降低购买门槛，再通过社群团购/接龙或者直播的形式达成批量转化成交的目的。

利用这种转化模型去做增长的案例：

- 长投学堂：位于上海，用“轻课程”撬动小白理财人群，客单价约2000元，微信生态投放ROI优秀，2018年长投学堂获1亿元A轮融资。
- 核桃编程：聚焦素质教育数理思维培训，采用年课打法，单月营收达到3200万。
- 斑马AI（包括斑马思维，斑马英语，斑马语文）：2019年整体营收超十亿元，2020年3月峰值超过3亿元，主要针对学龄前或启蒙阶段儿童。

## 二、社群转化模型的优势和劣势，怎么判断自己业务是否适合？

看了上面三个案例，是不是觉得社群转化模型太牛了，赶快用起来就起飞了😊

朋友们冷静一下，咱们还是得从业务出发，根据自己的业务来判断是否合适。

我先讲下我司的例子，我司主要是资格证业务，一建二建，财会经济，拥有系统成熟的电销团队。初期有考虑过用电销去打数据分析课程的转化，但有个非常现实的问题：数据分析是一个技能课程，没有考证类那么刚需，在没有充分信任及意向度的时候，很难转化，甚至大部分用户根本不鸟你。当时我们就有个判断：往后愿意接电话接受营销的用户肯定越来越少；

另外当时正好接触了风变的社群转化模型【可以看上篇： 】就觉得这种模式可以试一试，当时没太深入研究，测试之后拿到了结果，算是误打误撞，有点运气~

现在回头来看，总结有以下三个点是适合走社群转化模型的：

- 用户基数大，需求不刚但相对集中【核心】

我们业务的目标用户群体庞大，大部分都是职场白领蓝领，需要用到Excel表格完成日常工作的职员，这批用户保守估计国内有一个亿。这些人群学习技能课程本身非刚需，但需求相对集中，明确需要学习Excel的使用和数据分析思维提升自己的工作效率。利用社群转化模型能批量去扩需，提升他们的意向，高效批量转化，节省人力成本。

- 成交产品客单价在 1k~5k 之间

如果客单价低于 1k，容易造成账划不来。销售的人力成本+社群的流量成本，如果客单价过低，转化崩的情况就会高成本+低转化，双倍伤害，一定要算清楚人效比。如果客单价高于 5k，特别是过万的，纯靠社群拿下的难度太大，可以采用社群转化和电销转化（或1v1会议）结合的方式。

- 能通过社群运营增强用户体验

业务的核心产品，我们的0基础入门5天数据分析体验营，可以依赖群体互动来提升体验，如群内互动答疑，营造学习氛围和群体归属感等；如果产品无需通过社群提供额外价值，比如1v1的试听课，听一节课用户体验就很好，就可以决定是否买单的，这种也不需要。

## 三、如何设计社群转化模型？

ok，其实设计一个社群的转化模型，其实并不复杂。

先说结论：社群转化模型 = 体验课内容 + 体验营SOP + 数据监控体系。

接下来我会分别讲解这三块该怎么弄，给大家可操作的方法。

### 1、如何设计体验课内容

内容这块非常重要，我将内容分为课程主题和上课形式；不用瞎想怎么搞，明确咱们的用户和正式课产品，以终为始去倒推，一切就好办了。

以数分体验营举例，我们的用户是28~45岁的职场蓝领，想通过学习Excel表格和数据分析提升他们工作效率和业务能力，最后能升职加薪。那我们的正式课的课程内容主要就包含提效、汇报、分析这三个模块。另外这些职场上班族常常加班，就算准点下班回家，这个年纪大部分都有家庭，而且我们的用户女性占比6成，回到家也没那么多时间上直播，所以我们正式课还是以录播的形式为主，直播作为辅助，作为答疑或者分享直播。

ok，综合以上两点，我们体验营就是个浓缩版的正式课。内容选定：提效、汇报、分析三个主题各一节录播课，第四节课安排助力如何利用这项技能升职加薪的营销直播，这就是我们的基本框架。后续可以视运营节奏去增加内容，如扩需晚分享，或配合截单再加录播、直播课等。具体每节课的内容如何设计这里就不展开了，在做营销课内容这块有更多问题，可以联系鱼丸链接我聊下。

有些细节点分享一下：

- ①体验课一定要让学员收获感很强，所以大部分体验课其实都是正式课里面的精华内容。
- ②体验课要切中大部分用户的痛点，我们实际操作中会有些用户觉得我们的课程太小白，安抚并引导我们后续有高阶课，留联针对性聊成交即可；切勿为了这些特殊用户大改我们的内容结构。
- ③体验课录播时长控制在30分钟以内，适合通勤、饭点等碎片时间都能看。太长（40分钟以上）不好，学员会有压力，且时长越长，完课率会越低；太短（10分钟以内），我们的技能课都很难讲完一个完整的案例…也会让人觉得内容较水，跟短视频一样了。

### 2、如何设计体验营SOP

体验营SOP是根据我们体验营整体的学习节奏来，一般分为导量期、服务期、转化期。

我们把刚刚定下来体验课课程内容和形式放进去，这样一个表：

| 阶段 | 导量期 (Day0) | 服务期 (Day1) | 服务期 (Day2) | 服务期 (Day3) | 转化期 (Day4) | 转化期 (Day5) | 转化期 (Day6) |
|:---|:---|:---|:---|:---|:---|:---|:---|
| **日期** | Day0 | Day1 | Day2 | Day3 | Day4 | Day5 | Day6 |
| **课程内容** | 接量 | 提效 | 汇报 | 分析 | 升职加薪（直播） | VBA（加餐） | 逼单 |
| **作业** |  | Day1作业 | Day2作业 | Day3作业 | 测评 | 感悟 |  |
| **服务动作** | 接量添加、拉群开营 | 督学答疑、发福利 | 督学答疑、发福利 | 督学答疑、发福利 | 引导写测评、参与直播 | 引导写感悟 | 赠结营证书 |
| **转化动作** | 私聊探需 | 扩需甚至早鸟成交 | 社群产介 | 群发转化 | 班会产介 | 私聊截单 | 活动逼单、解散/团购 |
| **销售朋友圈** | 铺设品牌、学习好评 | 铺设品牌、个人IP、学习好评 | 个人IP、学习好评 | 课程宣发、学员好评、老师赠言 | 学员好评、优惠活动、学员好评 |  |  |
| **后续增加/分享内容** | 开营班会 | 晚分享 | 晚分享 | 晚分享 |  |  | 结营班会 |
| **数据目标** | 添加率、进群率 | 看课率、完课率、作业率 | 看课率、完课率、作业率 | 看课率、完课率、作业率 | 出勤率、下单率、支付率 | 意向回复率 | 截单成功率、转化率 |

别忘了我们需要让用户输出的东西：作业、测评、感悟等。根据以上信息定义我们销售具体要做的事情就很简单了。

在导量期，销售就是要做好接量和添加，让用户进来的第一印象好，清晰什么时间开始学习，提供开营前要准备的资料包，预习（后面做成了开营必读的短视频）。

在服务期，销售主要做好督学和作业的答疑。学习是反人性的，每天早中晚都需要催课，私聊和群内公告去督学，还可以用作业福利去作为钩子引导他们看课完课交作业，让学员多在课程上付出，我们的转化就会越有收获。

【提供一个数据给大家，完成三天作业的用户我们的转化率可以做到20%，客单价3000，而完成0天作业的仅1~2%，所以差距是巨大的。】另外，学员在实际完成作业过程中，会遇到一些卡壳的地方，这个时候会来问带班老师，带班老师辅导他们解决问题之后，这个过程也会增加双方之间的信任，为后续转化埋下基础，所以有些作业我们会设置一些小坑，增加学员与老师之间的沟通频次。

在转化期，我们的第四天直播是最为重要的，所以关键是引导学员来听直播，直播间直接下单批量转化。后续Day5、Day6的设置其实都是为转化逼单了，Day5设置结营班会，Day6设置解散群或者团购等活动。

### 3、搭建数据监控体系

做好了体验课内容及SOP，能让我们更好的批量复制和培训销售了。但规模放大之后，管理非常重要，利用数据监控可以辅助我们很好的去监控和管理销售团队。还有一个点非常重要，咱们的模型是要持续优化迭代的，利用这个数据可以对结果有一个较为客观的评价，针对每次的动作和数据变化去验证我们的迭代是否有效。

ok，搭建数据监控体系，具体要看哪些数据呢？

在销售角度，最核心的结果数据是转化率、GMV。项目操盘的角度来说关注GMV、单客成本、ROI、利润，这里我们不过度展开，主要看社群转化模型的转化率数据。

但只看转化率肯定是有问题，这个数据已经是一期训练营跑下来的结果了。我们抓管理肯定要抓过程，所以要拆解中间的一个个过程指标，最后通过这样一个过程数据的监控或者优化，得到一个比较好的转化率。

前面我们的运营流程已经确定了，这里的过程数据指标就可以较简单的定义了。

在导量期，最重要的加好友，也就是添加好友数量、添加率、添加好友数量/流量数量（流量是分配给该销售的例子数量），这个指标非常重要，流量成本这么贵，你懂得；还有个指标是进群率，我们可是社群转化模型，不进群，转化的池子数量基数少，转化的压力也会变大。

在服务期，最重要的是让学员看课、完课、做作业，也就是看课率、完课率、作业率这三个指标。前面也说过，完成三天作业的用户转化率可以做到20%，同渠道流量，看销售的服务水平在这里就会显示差距，销冠的三天作业率就是非常稳定且在一线水平。

在转化期，最重要的是让学员参加周四当晚直播，所以这里面的直播出勤率很重要；相关的直播对应的数据，峰值率、留存率、下单率、支付率，周五的群发的意向回复率，这些指标都尤为关键，对应的动作测试和迭代也是最多的。我们这块做得非常细，甚至每一分钟流失多少人我们都做了分析。

当以上的数据同渠道都跑了一段时间之后，就会有一个大概的基准值，可以在基准值之上设置数据目标，让销管带着销售往前冲。我们后续每做一些动作迭代也可以记录下来数据的变化，验证动作是否有效。

这里可以关注两个分析思维方法，一个是漏斗分析思维，我们的运营目标就是让用户尽可能的沿着我们设计的路径完成体验，最终购买正价课，在每个环节的漏斗都可以思考还有啥方法让流失的人更少；

第二个是筛选思维，我们需要销售将更多的时间精力放在高意向的用户身上，在紧张的转化期，每个销售的时间精力是固定的，优先将完成三天作业的用户成功转化是保持业绩稳健的关键，切忌放太多时间在没完成作业的用户身上。
