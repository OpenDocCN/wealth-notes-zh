还是不免费了，反正只有少数做生意和有阅历的人才能看明白。

正文外卖是流量的入口，就算不挣钱，也很高频。电商为什么要做小视频呢？无非也是提高打开的频次和增加用户停留的时间。只有在流量入口打开的频次越高，停留的时间越长，才可能转化出更多的电商订单！

如果按这个逻辑，微信才是最应该做电商的平台，没有什么东西比微信打开的频次更高和停留的时间更长，微信确实尝试过，只不过效果不明显，最后转为投资电商并为电商引流的策略！你不得不佩服腾讯的这个自我战略定位，因为腾讯自己是轻资产的互联网企，这个基因决定了腾讯不适合干苦力。

人能够发现自己的优势，也能发现自己的劣势并且守在自己的能力圈之内击球，这是非常难能可贵的品质！这里说一句题外话。一个企业的原始基因为什么决定后续的业务边界呢？原生家庭这个词已经普及了！原生家庭所说的就是人在成长过程中被雕塑的最初模型，不要小看了精神结构及自身与外部关系的最初模型，后续的一切都是对原生家庭这个基础模型的延伸！

大家都知道腾讯家的员工都比较有生活，甚至有点小资，为什么会形成这样的风格呢？一方面是基于做的事，另一方面就是基于最初的人。先说后面一个，当一个组织在创建之时，最初的人就决定了这个组织的基因，这种基因决定了这个组织的行事风格和能力圈层。这两者又决定了后续筛选人的侧重方向。如果筛选的人都偏向这个类型，就会在内部形成共振，继而在某个阶段叠加放大的发挥这个类型的优势，恰恰是这种成功的经验会促使整个组织在后续的筛选中强化这一偏好！

说完了后一个再说前一个。你最初做成功的事，会强化你在这个方向和这个形态上的投入。比如说你擅长做一件事情，你就会更喜欢做这种事情，你会把时间精力更多的花在这件事情上，即而形成一种循环叠加的优势。如果你有足够的阅历和精力，你就会轻易的发现这一点：当一个行业拼命内卷的时候，他们不会把资源投入到开拓新方向上，反而会把资源精力都投入到在原有的赛道上，争取比竞争对手更快半步。为什么宁可这样内卷也不改弦更张呢？在回答这个问题之前，可以轻易从个体的行为上得到的答案！假如一个人是做销售的，做了10多年，在行业内积攒了一定的人脉和客户，现在人工智能很火，你觉得这个做销售的人会去改行做人工智能吗？那他为什么不会呢？反而选择在一个更艰难的环境中试图挤出水来呢？因为能力圈层和路径依赖！

腾讯擅长做基于社交关系链的业务，这是一种轻资产的数字业务，能够轻易地获取海量的流量。但是，如果要把这些流量变现，则依赖于后端的落地。有些后端是数字化的腾讯可以轻易的吃下，即便之前暂时落后，也会最终占据绝对优势地位，比如说视频、支付、游戏等。如果落地的后端不是数字化的，而是一种更加传统、更加碎片化、更加不成体系的重资产形态，腾讯就搞不定了。这种情况下，选择与别人合作，比自己下场更好。腾讯经过一些年的探索之后，更加坚定了这一点，有自知之明的人，才是睿智和高明的决策者！不需要大喊大叫，也不需要站在聚光灯下，润物细无声的做对了正确的抉择。

长期以来腾讯的主要对手是阿里，阿里的最大贡献与其说是电商，不如说是帮助中小商家完成数字化升级改造。不管你喜欢阿里还是不喜欢阿里，你都不能否定阿里的这个历史贡献，这个贡献即便再过50年，回头看，依然举足轻重，并且对整个社会和时代都有巨大的价值！阿里不是做了某种业务或者挣了多少钱，而是把中国这个社会中大量散落的群体有序的组织了起来，完成了一次整体性的数字化升级。许多讨厌阿里的人说，正是因为这种数字化的升级，让自己的街边小店没有生意，这是马车夫对汽车的诅咒，并不改变历史进程。

任何社会最难的事，都是把散乱落后无序的基础，以一种更先进有效的方式，提高到一个更高效的层次，这个东西被称为基础设施升级。用过计算机的人都明白，软件的增减，可以扩展应用的功能，但是，操作系统和硬件的升级，才能从根本上决定应用的水准！你拿一个386的老机器装个 DOS 系统试试，就会发现现在的所有软件都没法用，而能够用的那几个老软件，在目前的这些业务需求上没有任何用处，根本满足不了需求。从这个角度你就能理解1949年新中国成立的历史价值和伟大之处，1949年完成了硬件的升级，并且把系统重新设计了，后续的那些应用才能跑在这样的硬件和操作系统上。

当西方的技术资金和产业逐渐向亚洲转移的时候，其实是先到达东南亚，最后才到达我们中国这里。上世纪90年代，亚洲四小龙，就是最早承接这种转移的社会，为什么今天他们依然和40年前没啥区别，而我们中国却日新月异呢？因为我们中国的系统升级了，硬件升级了，亚洲四小龙还是以前的硬件和系统，能够做的微调非常有限！你用15年前的苹果机装现在的软件一个都用不上，如果你用15年前的软件来处理现在的需求，你会发现慢的要死，根本没法用，放大到一个社会这样的模型也是如此。

言归正传，阿里完成了一次对中小卖家和生产商数字化升级改造，从而大幅的提升了我们中国这个社会的生产效率和贸易效率，接下来，我们这个社会将从由生产主导转向消费主导，那么，谁能完成生活场景中各要素的数字化改造和升级，谁就是新时代的功臣。阿里在完成那次使命的时候是不是以垄断者的角色出现呢？必须是！当年的秦皇汉武以及教员毛泽东在完成社会基础结构升级的时候也是以这种角色出现，因为只有这种垄断者角色才能完成这样的事情！

消费场景的数字化重构，并将人和各要素有序的纳入到一个系统之中，这件事比上一次阿里对于卖家和卖家的数字化改造更加繁琐也更加艰难，因为利润空间更有限！

阿里最开始地推开拓的时候非常艰难，因为各个生产厂家对这种新事物不熟悉，不愿意为之付出成本，但不管怎么样对接的都是厂家，并且有一定的经济实力。如果新的模式能够带来新的业务，厂家就会愿意投入。

然而，当下所要面对的生活场景中的各个环节，都是没实力的对象。不管是开快餐店的还是跑腿的都更加的没有定力，更不会因为你的这种改造能够立竿见影获得超出预期的回报！这与当年阿里去改造生产商不同。

一个工厂，哪怕是你能够从线上给他带来一两个订单，这个工厂都会愿意为开拓一种新的未来而投资，愿意主动去配合你完成自我调整，比如说组建一个新的部门去对接，但是一个小店本来就有小本经营，无法做任何改变也不愿意支付任何成本去尝试一个不能立竿见影获得回报的事，所以，这一次会更难！

谁来完成这个使命，我们先不要下结论，但可以肯定的是：绝不会是红衣服京东外卖那家。为什么呢？现在的外卖方式属于上一个时代的补丁！为什么说是补丁呢？上一个时代的主题是以电商的形式完成对于买家和生产商的数字化改造和升级，从而实现整个社会的生产和贸易效率的大幅提升，完成这个使命的主导者是阿里，但阿里在完成这套系统升级的时候，遗漏了一个 bug，虽然搞了一个菜鸟物流网络，但是，没有能够下沉，从而让整个交付链条缺了一环。

红衣服京东平台背后的投资人看到了这个 bug，找了一个愿意干重活的人来打上这个补丁，所以，红衣服京东平台不管办公室的瑜伽做的多好，其核心依然是在风雨中的那群送快递的物流汉子，但送快递的物流汉子是劳动密集型，这种优势不可持续，优势不是在主干物流，而是末端物流，这种先发优势会随着时间的推移逐渐被拉平，一旦模式成熟了，就会被快速复制，最后就变成了谁投入的多，堆的量更大，谁就在一个成熟的模式中有规模优势。除此之外，末端物流的后面还有最后 1 公里，这 1 公里就是社区生活场景的快速响应，今天大家抢的就是这个。

在整个交付环节中，主干物流由国家主导，末端物流就是从机场和火车站以及长途汽车转运站出来到前置仓或者门店的这一 段。经过这 10 多年的建设、优化和博弈，基本上已经成体系了，形成了一个公共化的设施，有些物流转运站和仓储体系甚至是官方投资 建的，租给大家用，所以大家在这个层面的差异化越来越小。接下来是缩小范围，在更小的生活场景内完成消费和交付的闭环，不再是上一个时代那样：用户在线上下单，平台通知生产商从仓库发货，经过主干物流和末端物流的配送，几天后送到用户手上，这个模式在以前很先进，现在的人已经无法忍受。现在的需求变成了：我想要一个解决当下问题的方法，不管要用到什么样的商品和服务，要有一种方式立刻给我解决，这就是一个生活场景的综合性需求。解决方案就变成了：在这个生活场景之中存在可以解决这个问题的资源，然后有一个平台来完成调度和所有相关的善后问题，中途需要卖点什么只是解决问题过程中顺带的，而且需要的东西在周围必须要有。

这是一个更难的事：你不可能为每一种需求建各种各样的直营店，你只能生活场景中本来就存在的那些店铺仓库生产商卖家，以一种系统性的方式连接起来并完成调度，而调度的有效性不在于程序，而在于利益。大佬文集圈生活场景中系统性方式的有序构建非常复杂，却又没有那么高的利润来驱使各要素参与，在这个过程中所有的摸索投入经验教训，都是护城河。什么意思呢？就是你踩过的坑别人也要踩一遍，你花的时间，别人也要花这么多时间。在计算机行业里面有一个著名的典故——无比低调的 C++ 之父曾经罕见的发表了一个看法：五十年来，有无数的编程语言，试图挑战 C++ 的地位，他们一开始都指责 C++ 过于复杂，但他们忽略了一件事 C++ 的应用囊括计算机工业的所有领域，当那些试图挑战的新语言也试着去解决更多问题的时候，它们要么挑战失败，要么就变得越来越像 C++。

三年前在写下《2022 年 0411：算账！》的同时也买了 500 万人民币的卢布，这里记录下形成判断的思路过程是给以后的自己看的。就如同今天再回头看《2022 年 0411：算账！》中的内容，发现三年前的自己虽然在战略方面的判断是对的，但在战术上是有出入的。三年前我认为俄军在去摧毁乌军后勤生产的基础设施并支解乌军对俄空天军的高空威胁之后，会集合大量的部队以装甲集群的方式配合空天军推进，俄军采用的不是装甲集群，而是无人机配合战术小队，以渗透式的方式进行推进。这是对战术变化的认知不足，源于深受苏军战法和体系的惯性影响，这种回过头的检讨是有意义的，但这种检讨需要时间，让尘埃落定之后才能回过头去审视判断的形成思路。
